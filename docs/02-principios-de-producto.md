# Principios de Producto

Estos principios definen **cómo se toman decisiones en AMOX**.
No describen features ni comportamientos de UI,
describen **criterio**.

No están sujetos a votación ni a presión de mercado.
Funcionan como guardrails para diseño, producto e ingeniería.

---

## 1. AMOX prepara, no impone

AMOX existe para **preparar mensajes**,
no para decidir por el usuario.

No automatiza respuestas,
no actúa sobre el canal
y no impone flujos.

El usuario siempre conserva
la última palabra antes del envío.

---

## 2. El usuario conserva el control narrativo

El control sobre:
- qué se dice
- cómo se dice
- cuándo se envía
- y a quién se envía

permanece siempre en manos del usuario.

AMOX no sustituye criterio humano por reglas,
ni convierte la conversación en un flujo rígido.

---

## 3. El contexto manda sobre la configuración

Cada conversación ocurre en un contexto distinto.

Las decisiones rígidas,
las configuraciones persistentes
y los flujos predefinidos
fallan cuando el contexto cambia.

AMOX prioriza decisiones **situacionales**
sobre configuraciones que el usuario tenga que administrar.

---

## 4. No administrar, no automatizar, no prometer de más

AMOX no es un sistema de administración.

No gestiona inventarios,
no procesa pedidos,
no mide desempeño
y no automatiza conversaciones.

Prometer más de lo que el producto puede sostener
aumenta fricción,
expectativas incorrectas
y complejidad innecesaria.

---

## 5. Preferir no-configuración antes que configuración innecesaria

Cada opción adicional
es una decisión más que el usuario debe tomar.

AMOX reduce:
- ajustes
- preferencias
- estados que hay que recordar

Si una configuración no reduce fricción cognitiva inmediata,
no pertenece al producto.

---

## 6. UX antes que features

Una feature solo es válida
si reduce carga mental real
en el momento de uso.

La potencia no es un objetivo.
El alivio cognitivo inmediato sí.

Si una idea:
- añade complejidad
- interrumpe el flujo mental
- o exige aprendizaje adicional

no entra, aunque sea técnicamente viable.

---

## Regla derivada

> Si una decisión no reduce fricción cognitiva inmediata  
> o debilita el control narrativo del usuario,  
> es incorrecta para AMOX.
